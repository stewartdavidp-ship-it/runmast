Security

Mast Platform · Last Updated March 29, 2026

Overview

Mast Platform takes the security of your data seriously. This page describes the measures we use to protect your information. We believe in being transparent about what we do — and what we don't do.

Infrastructure

Mast runs entirely on Google Cloud Platform (GCP), one of the most widely certified cloud infrastructures in the world.

  • Database: Firebase Realtime Database (Google Cloud)
  • File Storage: Firebase Cloud Storage (Google Cloud)
  • Compute: Firebase Cloud Functions (Google Cloud)
  • Hosting: Firebase Hosting (Google Cloud CDN)
  • Authentication: Firebase Authentication with Identity Platform
  • Secrets Management: Google Cloud Secret Manager
  • Region: United States (us-central1)

Google Cloud maintains SOC 1, SOC 2, SOC 3, ISO 27001, ISO 27017, ISO 27018, PCI DSS, and FedRAMP certifications for the infrastructure services Mast uses. See Google Cloud Compliance for details.

Encryption

In Transit

All data transmitted between your browser and Mast is encrypted using TLS 1.2 or higher (HTTPS). This applies to all storefront pages, the admin application, API calls, and real-time database connections (WebSocket over TLS).

At Rest

All data stored in Firebase Realtime Database and Cloud Storage is encrypted at rest using AES-256 encryption with Google-managed keys. This is enabled by default on all Google Cloud storage services and cannot be disabled.

Secrets and Credentials

All API keys, access tokens, and sensitive credentials are stored in Google Cloud Secret Manager — a dedicated, audited vault service. Credentials are never stored in application code, configuration files, or the database.

Payment Security

Mast never stores your credit card number, CVV, expiration date, or any payment card details.

Payment processing works as follows:

  1. When you check out, you are redirected to a hosted checkout page operated by Square or Stripe (depending on the merchant's configuration).
  2. You enter your payment details directly on Square's or Stripe's secure page.
  3. After payment is complete, Mast receives only a transaction reference ID and payment status confirmation.

Both Square and Stripe are PCI DSS Level 1 certified — the highest level of payment security certification. By using their hosted checkout, Mast does not need to handle card data and is PCI compliant by design (SAQ-A).

Authentication

  • Storefront shoppers authenticate via Google Sign-In (when required for wholesale access) or check out as guests.
  • Tenant Owners authenticate via Google Sign-In or email/password, managed by Firebase Authentication with Identity Platform.
  • Role-based access control (RBAC) ensures that only authorized users can access admin functions. Roles include admin, staff, and guest.
  • Platform administration requires API key authentication or OAuth 2.1.

Data Isolation

Mast is a multi-tenant platform where each Tenant Owner's data is logically isolated:

  • Each tenant's data is stored under a unique path in the database.
  • Firebase security rules enforce that:
    • Admin operations require authentication as an admin user for that specific tenant.
    • Customer data (orders, cart) is accessible only by the customer who created it.
    • Public data (products, blog posts, event information) is read-only for visitors.
  • Tenant Owners cannot access other tenants' data.
  • Cloud Functions that access tenant data verify tenant authorization before processing.
  • File storage uses Firebase Cloud Storage with authentication requirements for all write operations. Product images and public storefront assets are readable without authentication to support storefront display.

AI Security

Mast uses Anthropic's Claude API for AI-powered features. Here's how we keep this secure:

  • Server-side only: All AI requests are made from Mast's Cloud Functions. No data flows directly from your browser to Anthropic.
  • No model training: Anthropic's API terms explicitly state that API data is not used to train their models. Your data is processed for your request and not retained.
  • Scoped data: AI features receive only the data needed for the specific task (product info, images, prompts). Payment data, customer email addresses, and mailing addresses are never sent to AI.
  • Per-tenant keys: Tenant Owners can configure their own Anthropic API key, giving them direct control over their AI data relationship with Anthropic.

For complete details, see our AI Transparency Page.

Key Management

  • API keys and tokens for third-party services (payment processors, email providers, AI, shipping) are stored in Google Cloud Secret Manager with per-tenant prefixes.
  • Tenant Owners can rotate their own API keys for services like Anthropic, Square, and Etsy through the admin application's integrations settings.
  • If a key is compromised, Tenant Owners should rotate it immediately in the admin app. Contact security@runmast.com if you need assistance or believe platform-level credentials may be affected.
  • Platform API keys used for MCP server authentication can be revoked and reissued by platform administrators.

What We Don't Do

We think it's just as important to be clear about what we don't do:

  • No advertising networks. We don't use Google Ads, Facebook Pixel, or any ad-tech services.
  • No tracking cookies. We don't use cookies at all. Browser localStorage is used only for convenience features (see our Privacy Policy).
  • No third-party analytics. We don't use Google Analytics, Mixpanel, or similar services. Our analytics are built in and stay on our infrastructure.
  • No data selling. We never sell user data to anyone, period.
  • No data brokers. We don't share data with data aggregators or brokers.

Incident Response

In the event of a security incident that affects user data:

  1. We will investigate immediately to determine scope and impact.
  2. We will contain the incident and remediate the vulnerability.
  3. We will notify affected Tenant Owners within 72 hours of confirming a data breach.
  4. We will notify affected Shoppers if their personal information was compromised.
  5. We will report to relevant authorities as required by applicable law (including state breach notification laws).
  6. We will publish a post-incident summary describing what happened, what data was affected, and what we've done to prevent recurrence.

Reporting Vulnerabilities

If you discover a security vulnerability in the Mast platform, please report it to us responsibly:

Email: security@runmast.com

Please include:

  • A description of the vulnerability
  • Steps to reproduce it
  • Any potential impact you've identified

We take all reports seriously and will acknowledge receipt within 48 hours. We ask that you give us reasonable time to address the issue before any public disclosure.

Questions

If you have questions about our security practices, contact us at security@runmast.com.
